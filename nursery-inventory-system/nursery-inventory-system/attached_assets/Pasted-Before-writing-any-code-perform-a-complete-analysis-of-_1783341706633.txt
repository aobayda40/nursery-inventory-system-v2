==================================
WORKFLOW REQUIREMENTS
==================================

Work in small logical milestones.

After completing each milestone:
1. Verify it works.
2. Run the build.
3. Fix any errors immediately.
4. Commit the changes with a clear Git commit message.

Do NOT leave the repository in a broken state.

If the Replit session is about to end or credits are running low:

1. Finish the current task.
2. Run pnpm build.
3. Fix any remaining errors.
4. Commit all completed work.
5. Push every commit to GitHub.
6. Update ROADMAP.md with completed and remaining work.
7. Update replit.md with implementation notes, important decisions, setup instructions, known issues, and the exact next task to continue.
8. Provide a concise handoff summary explaining:
   - What was completed
   - What was changed
   - What database changes were made
   - What APIs were added
   - What frontend pages were added
   - Any remaining issues
   - The exact next development step

Never stop with uncommitted work.

Never leave partially implemented features if they can be completed within the remaining session.

Always make GitHub the latest working version before finishing.
